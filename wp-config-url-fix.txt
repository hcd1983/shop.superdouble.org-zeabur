===== WordPress URL 強制設定 =====

如果上面的修復腳本無法解決問題，請在 wp-config.php 中加入以下程式碼
（加在 require_once(ABSPATH . 'wp-settings.php'); 這行之前）：

// 強制設定網站 URL（請修改為你的實際網址）
define('WP_HOME','http://localhost');
define('WP_SITEURL','http://localhost');

// 如果是 HTTPS，請改為：
// define('WP_HOME','https://你的網域');
// define('WP_SITEURL','https://你的網域');

// 關閉自動更新重新導向
define('WP_AUTO_UPDATE_CORE', false);

===== .htaccess 修正 =====

請將 .htaccess 中的 Wordfence 路徑更新：

原本：
php_value auto_prepend_file '/home/atomdpc3/shop.superdouble.org/wordfence-waf.php'

改為你的實際路徑，例如：
php_value auto_prepend_file '/Users/deanhuang/Desktop/shop.superdouble.org/shop.superdouble.org/wordfence-waf.php'

或者暫時註解掉 Wordfence 相關設定（在每行前面加 #）

===== 資料庫直接修改（如果有資料庫存取權限）=====

執行以下 SQL 查詢來更新 URL：

UPDATE wp_options SET option_value = 'http://你的新網址' WHERE option_name = 'home';
UPDATE wp_options SET option_value = 'http://你的新網址' WHERE option_name = 'siteurl';

===== 清除快取 =====

1. 清除瀏覽器快取
2. 使用無痕/隱私模式測試
3. 清除 DNS 快取：
   - Mac: sudo dscacheutil -flushcache
   - Windows: ipconfig /flushdns

===== 常見的重新導向來源 =====

1. 資料庫 wp_options 表的 home 和 siteurl
2. wp-config.php 中的 WP_HOME 和 WP_SITEURL
3. .htaccess 檔案中的重寫規則
4. 主題或外掛的設定（特別是 SEO 和快取外掛）
5. DNS 設定或主機商的重新導向設定
6. Wordfence 或其他安全外掛的設定
